> Companion to `AGENTS.md`. Read this before adding any user-visible string.

## Structure

```typescript
export type Lang = 'tr' | 'en';

export const translations = {
  tr: { key: 'Turkish value', ... },
  en: { key: 'English value', ... },
};
```

## Rules

1. **Always add to BOTH locales.** If you add a key to `tr`, you MUST add the same key to `en` (and vice versa). Missing keys cause `undefined` to render in the UI.

2. **Key naming convention:** camelCase, descriptive of meaning (not location). Good: `deleteAccount`, `verifyEmail`. Bad: `buttonText3`, `headerLabel`.

3. **No interpolation system exists.** For dynamic values, use template literals in the call site:
   ```typescript
   // In i18n.ts:
   movedTo: 'taĹźÄ±ndÄ±'        // 'moved to'
   // In App.tsx:
   showToast(`${t.movedTo} ${folderName}`)
   ```

4. **Do not put HTML in translations.** Plain text only.

5. **Error messages** from the API are in English (from the server). Only translate client-side error messages (validation, network errors).

6. **Verification flow is link-based.** Keys like `verificationCodeSent`, `verificationCodeSentIntro`, `checkEmailInstructions`, `checkVerificationStatus`, and `resendCode` currently drive Firebase email-link copy, even if some names still mention "code" for backward compatibility.
